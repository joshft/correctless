# Correctless — bundled fallback secret-redaction pattern set (OQ-007 / INV-013)
#
# Format: POSIX-ERE, one pattern per line. Lines beginning with '#' (after
# optional leading whitespace) and blank lines are ignored. Patterns are applied
# case-insensitively by scripts/redact-secrets.sh; every match of every pattern
# is replaced with the literal <REDACTED>.
#
# This is the SHIPPED minimum starter set pinned by OQ-007. tests/test-redact-
# secrets.sh enumerates every active line here and asserts a matching sample is
# redacted, so coverage can never lag the shipped set (AP-031 guard). Patterns
# must be ERE that GNU `grep -E` / `sed -E` accept — no PCRE-only constructs
# (no `\d`, no `(?i)` inline flags; case-insensitivity is handled by the script).

# AWS access key id
AKIA[0-9A-Z]{16}

# Generic high-entropy secret/token/password/api_key assignment (>=16 chars)
(secret|token|password|passwd|api[_-]?key)[[:space:]]*[:=][[:space:]]*['"]?[A-Za-z0-9/+=_-]{16,}

# Private key PEM header
-----BEGIN [A-Z ]*PRIVATE KEY-----

# JSON Web Token (three base64url segments). Spec OQ-007 pins {8,} per segment
# (the spec-pinned floor — see .correctless/specs/cchores.md OQ-007). The
# enumeration guard in tests/test-redact-secrets.sh synthesizes a sample with
# >=8 base64url chars per segment so coverage tracks this exact shape, and a
# dedicated INV013-JWT-FLOOR assertion fails if the floor ever drifts off {8,}.
eyJ[A-Za-z0-9_-]{8,}\.[A-Za-z0-9_-]{8,}\.[A-Za-z0-9_-]{8,}

# GitHub token (ghp_/gho_/ghu_/ghs_/ghr_ + 36+ chars)
gh[pousr]_[A-Za-z0-9]{36,}

# Slack token (xoxb-/xoxa-/xoxp-/xoxr-/xoxs-)
xox[baprs]-[A-Za-z0-9-]{10,}
