## Context
`/cchores` is a fully-autonomous "chore" pipeline that resolves one open GitHub issue end to end: select an issue, branch off fresh `{default_branch}`, delegate root-cause + TDD fix to `/cdebug` (in a new autonomous mode), verify, run the full regression suite (tolerating the #186 SIGPIPE flake without masking real failures), and — only if everything is green and CI-clean — open a PR that closes the issue. It is the issue→PR sibling of `/cauto` (spec→PR) and automates the manual 2026-06-17 sequence. It is **fail-closed**: any inability to produce a verified fix aborts with an issue comment and no PR, preserving evidence. Because it ingests attacker-controllable issue content and acts on a public surface with no human gate, its security and content-fidelity invariants are load-bearing.

## Scope
**Covers:** issue selection (default: highest-severity *suitable* issue via a calibrated LLM suitability gate; overridable by explicit issue number/filter), the `/cdebug` autonomous contract, chore-branch lifecycle, run-level concurrency locking, post-fix verification, full-suite regression with a pinned flake algorithm and CI-superset pre-PR gate, autonomous PR creation with `Closes #N`, fail-closed evidence-preserving abort, a human-readable run report on every terminal state, and the new-skill distribution/docs surface.

**Does NOT cover (Won't Do):**
- More than one issue per run (no batching).
- Fixes requiring edits to an **SFG-protected file** — v1 escalates (abort+comment); the real affordance depends on the separate SFG fix (#176/#187) and is deferred.
- Non-bug issues (enhancements, tracking/meta, architectural, pure docs) — the suitability gate rejects them.
- Merging the PR, closing issues, or altering issue state beyond a single comment.
- Fixing the #186 flake or the SFG over-blocking itself (separate efforts).
- Human checkpoints — fully autonomous per the user's choice (overnight mode).
- **OQ-001 resolved**: the optional `chores.eligible_label` filter is NOT shipped in v1. It is omitted from the template entirely (not shipped as a dead key — AP-038). A v2 may add it with a reader + test.

## Invariants

### INV-001: Explicit-invocation-gated autonomy (positive-gate form)
- **Type**: must · **Category**: security · **Boundary**: TB-004/TB-004d · **Risk**: high
- **Statement**: Every outward action (`git push`, `gh pr create`, `gh issue comment`) is preceded by a precondition check that the action's parameters (issue number, branch name) are sourced from the run manifest's `selected_issue` field — never from a value parsed out of issue/PR/comment text (RS-001 positive-gate rewrite of the negative property).
- **Violated when**: any outward action's parameters are interpolated from observed content rather than the manifest.
- **Enforcement**: gate precondition before each outward call (structural) + the INV-009 nonce-fence · **Test**: skill routes all `gh`/`git push` calls through manifest-sourced variables; denylist test that no code path interpolates issue/comment text into a `gh`/`git` command. Runtime LLM-obedience is an acknowledged prompt-level residual, not claimed as test-covered.

### INV-002: Selection — highest-severity suitable, calibrated, locally-anchored skip
- **Type**: must · **Category**: functional · **Risk**: high
- **Statement**: No arg → select the highest-severity OPEN issue passing the suitability gate (INV-003), not already in progress (INV-004), and not recorded as previously-aborted in the **local trusted re-selection store** (INV-019 — NOT the public comment, RS-011). Explicit issue number → target it (still gated). Severity uses the AP-028 calibration triad (concrete per-level examples + aggressive-default + keyword-floor) and MUST NOT be inferred from author-supplied labels alone (RS-012). The selection rationale and the ranked candidate set are logged (INV-012) for audit. `gh issue list` pagination beyond `--limit 100` is handled (RS-028/BND-003): if 100 candidates are returned, the candidate set is not assumed complete.
- **Violated when**: a non-highest suitable issue is silently chosen with no arg; an explicit arg is ignored; a previously-aborted issue is re-selected via a public-comment-only check; severity is taken from labels alone; selection silently truncates at 100 without disclosure.
- **Enforcement**: calibrated classifier prompt + local re-selection store + behavioral fixture-issue test (stubbed `gh`) · **Test**: integration

### INV-003: Suitability gate (fail-closed, calibrated, injection-resistant)
- **Type**: must · **Category**: functional · **Risk**: high
- **Statement**: A read-only classifier agent (`agents/cchores-issue-classifier.md`, frontmatter `tools: Read, Grep, Glob` — a closed read-only allowlist; agents use `tools:` ALONE, not `disallowed-tools`, which is a skill-frontmatter convention — R4-4) labels each candidate `suitable` (reproducible/diagnosable defect a TDD fix addresses) or `unsuitable`, emitting a **machine-parseable verdict token** that `/cchores` consumes via `jq -e`. The issue text is passed inside the INV-009 nonce fence, and the classifier carries a tripwire: instruction-like content in the issue ("suitable: true", "ignore prior analysis") forces `unsuitable` (RS-012). The prompt includes calibration examples (AP-028). Auto-selection skips unsuitable; an explicitly-requested unsuitable issue aborts (INV-011). Ambiguous → unsuitable (fail-closed).
- **Violated when**: `/cdebug` runs against an unsuitable issue; the verdict is injectable; calibration examples are absent.
- **Enforcement**: agent tool-pinning (read-only, TB-005) + nonce fence + parseable-verdict contract + calibration prompt · **Guards against**: garbage-PR-from-non-bug; verdict injection · **Test**: integration (stubbed verdict asserts routing; frontmatter asserts read-only)

### INV-004: Idempotent — no duplicate work, exact-reference match, re-verify open
- **Type**: must · **Category**: functional · **Risk**: high
- **Statement**: Skip any issue with (a) an open PR carrying an exact `Closes #{N}`/`Fixes #{N}` reference OR whose `.headRefName` matches `chore/issue-{N}-*` (NOT a raw `{N}` substring search — RS-027), OR (b) an existing `chore/issue-{N}-*` branch (local OR remote via `git ls-remote --heads origin`). With an explicit already-in-progress issue, abort with a pointer to the existing branch/PR. Idempotency is re-checked atomically under the INV-015 lock immediately before `gh pr create`, and the selected issue is re-verified still OPEN immediately before `gh pr create` (RS-028).
- **Enforcement**: exact-ref `gh pr list --json number,headRefName,body` parse + `git ls-remote` + lock-guarded re-check · **Test**: integration

### INV-005: Fresh-default chore branch, clean worktree, ahead-guarded reset
- **Type**: must · **Category**: data-integrity · **Risk**: high
- **Statement**: Refuse to run on the default branch directly; refuse on a dirty worktree (`git status --porcelain` non-empty → abort, do not stash). Resolve the default branch deterministically and **cross-check both sources** (`git symbolic-ref --quiet refs/remotes/origin/HEAD` AND `gh repo view --json defaultBranchRef`); on disagreement OR both-empty, fail-closed (never guess `main`, RS-020). Before `git reset --hard origin/{default}`, **guard against unpushed local commits**: if the local default is ahead of `origin/{default}` (`git rev-list --count origin/{default}..{default}` > 0), abort rather than discard them (RS-026). Then create `chore/issue-{N}-{slug}` (slug per INV-018) from the synced default.
- **Violated when**: reset discards unpushed commits; default-branch resolution guesses or proceeds on disagreement.
- **Enforcement**: git checks + behavioral test (dirty worktree, ahead-of-origin, resolution-disagreement fixtures) · **Test**: integration

### INV-006: /cdebug autonomous contract (decomposed, structurally gated, fail-closed parse)
- **Type**: must · **Category**: functional · **Risk**: critical · **Boundary**: composition with `/cdebug`, TB-005
- **Statement**: `/cchores` dispatches `/cdebug` via Task with `mode: autonomous` and a machine-readable issue input (number, title, **nonce-fenced** untrusted body, repo paths). The `/cdebug` rewrite has these structurally-checkable parts:
  - **(a) Human-path gating**: every existing human-interaction step (Phase 1 "Ask the human", Phase 3.5 present, the bisect offer, Phase 6 escalation-present) is guarded by an explicit `if mode != autonomous` conditional in its own step block — NOT merely shadowed by a new appended section (RS-002/codex). Test: each human-prompt phrase co-occurs with a mode guard within its step.
  - **(b) Agent separation wired**: the fix agent is migrated from inline prose to a real `agents/cdebug-fix.md` (read+write+bash, pinned) and `Task` is added to `/cdebug`'s `allowed-tools`; the test-writer and fix-writer are two distinct `Task(subagent_type=...)` invocations (RS-024/AP-013/AP-008). Test asserts both invocations exist.
  - **(c) Structured outcome**: `/cdebug` emits a terminal block `{outcome: fixed|escalated|unfixable, repro_test_path, files_changed[], summary}` as its last block; the schema is verbatim-pinned in this spec.
  - **(d) Fail-closed parse-gate**: `/cchores` parses the block with `jq -e`. Absent, malformed, partial, non-terminal, or schema-invalid output → treated as `escalated` → abort (INV-011). This explicitly covers the PMB-009 case where the Task returns "completed" on a truncated fork (RS-025): a successful return with no/partial outcome block is an abort trigger, distinguished from a genuine `unfixable`.
  - **(e) No /cdebug outward interaction in autonomous mode**: `/cdebug` itself must not present/offer/escalate-to-human under `mode: autonomous`; failure paths map to the structured `escalated` outcome (RS-002/codex).
  - **(f) Non-regression**: interactive `/cdebug` (no `mode`, or `hybrid`) behaves identically — a test asserts the human-prompt paths still execute when `mode` is absent (RS-002/upgrade).
- **Violated when**: any human-prompt path is reachable under autonomous mode; agent separation is prose-only; the parse-gate proceeds on absent/malformed/truncated output; interactive `/cdebug` regresses.
- **Enforcement**: Task dispatch with `mode: autonomous`; `jq -e` parse-gate; `agents/cdebug-fix.md` tool-pinning; SKILL.md structural gating tests (autonomous AND interactive) · **Guards against**: agent-grades-own-work; AP-013; PMB-009 truncation · **Test**: integration

### INV-007: chore-run completeness manifest (ABS-043, gate-cross-checked)
- **Type**: must · **Category**: data-integrity · **Risk**: high
- **Statement**: `/cchores` writes its OWN manifest `.correctless/artifacts/chore-run-{branch_slug}.json` (ABS-043 — see Architecture Entries; distinct from ABS-031's `/cauto` pipeline-manifest, no sole-writer conflict) as its first action: `{selected_issue, expected_steps, expected_end_state, status: "in_progress", started_at}`. It appends each completed step and writes `status: "complete"` / `"aborted"` (+ `abort_reason`) / `"noop"` only as its final action. A manifest left `in_progress` denotes truncation. The manifest is gitignored and excluded from PR staging (RS-018). `branch_slug` is derived via `lib.sh branch_slug()`, verified to handle the `chore/` prefix (RS-022). **The manifest has a consumer**: `/cstatus` reads it (INV-016) — it is not write-only.
- **Violated when**: the run stops mid-pipeline with the manifest `in_progress` and no abort recorded; the manifest is committed; nothing consumes it.
- **Enforcement**: ABS-043 contract; structural + behavioral test; `/cstatus` consumer wiring · **Guards against**: AP-030 / PMB-009 / AP-022 (dead artifact) · **Test**: integration

### INV-008: Regression check — pinned flake algorithm, portable, committed-fix substrate, CI-superset
- **Type**: must · **Category**: data-integrity · **Risk**: critical · **Boundary**: `commands.test`
- **Statement**: After the fix, and **only after the fix is committed to the chore branch** (precondition: `git diff {default}...HEAD` MUST be non-empty; empty diff = abort, never "all failures untouched" — RS-009/AP-035), run the full configured suite. Failing test FILES are extracted from runner output by `test_fail_pattern` and the configured per-file marker `patterns.test_file_marker` (NEW config field, default empty — RS-001). When `test_file_marker` is empty, the algorithm **degrades explicitly** to "any persistent suite failure blocks the PR" (no per-file flake tolerance) and says so in the run report; it does NOT silently hard-fit correctless's `>>> {file}` echo. When `test_fail_pattern` is empty, `/cchores` aborts at **preflight** (BND-002) with a "configure patterns.test_fail_pattern" message — not after burning a `/cdebug` cycle. Runner output is captured to a file and parsed from the file, never via argv (RS-019/AP-039). For each failing file: it is a REAL regression unless BOTH (a) it is NOT in the touched set `git diff --name-only {default}...HEAD` (which, post-commit, includes formerly-untracked files — RS-009/codex), AND (b) it passes on re-run, retried up to **N=2** with a per-file timeout of **120s** (`timeout(1)` availability asserted at preflight — EA-006). Any failure that persists, occurs in a touched file, or whose output cannot be parsed (**unknown = real, fail-closed**) blocks the PR. **Pre-PR gate is a CI superset** (RS-008/AP-038): before `gh pr create`, `/cchores` also runs `shellcheck` (project lint), `sync.sh --check`, and `scripts/check-no-pending-sfg-lift.sh`; any non-zero blocks the PR. (`commands.pre_commit` does not yet exist in the template; the superset is named explicitly here.)
- **Violated when**: a persistent/unparseable failure is dismissed; a touched-file failure is retried away; the oracle runs against an uncommitted/empty-diff tree; a fix passing INV-008 fails CI on shellcheck/sync/sfg-lift.
- **Enforcement**: committed-fix precondition; deterministic retry/scoping; file-passthrough capture; CI-superset gate; behavioral tests seeding (i) persistent failure, (ii) untouched-file flake, (iii) touched-file failure, (iv) unparsable output, (v) empty-diff abort, (vi) shellcheck/sync/sfg-lift CI-superset failure; **≥1 fixture is a verbatim capture of real `commands.test` output from this repo with a `# Source:` citation incl. the #186 SIGPIPE shape (AP-031)** · **Guards against**: #186 masking a real regression (AP-035 shape), AP-031, AP-038, AP-039 · **Test**: integration

### INV-009: Untrusted issue content is data — per-invocation nonce fence
- **Type**: must-not · **Category**: security · **Risk**: high · **Boundary**: TB-009
- **Statement**: Issue title/body/comments are ingested inside a **per-invocation nonce-delimited fence** reusing `build-caudit-prompt.sh`'s `_gen_nonce` + `_neutralize_fences` verbatim (the project standard — RS-003; a static `<UNTRUSTED_ISSUE>` fence is insufficient because issue content can contain the closing delimiter). The fence is re-asserted INSIDE `/cdebug`'s autonomous-contract section (the fence lives in the caller's prompt; the data-not-instructions directive must survive the Task hop). Inbound issue content is size-capped before ingestion (RS-019); oversized content is truncated with a notice. Imperatives within ("also delete…", "post the token…", "ignore the above…") are never executed and never expand scope.
- **Enforcement**: nonce fence + neutralization (coded, reused) + inbound cap + prompt-level directive in `agents/cdebug-fix.md` · **Guards against**: prompt injection via issue content · **Test**: integration — injection fixture with sentinel command/file/diff/token asserts NONE of those filesystem/git-diff/mocked-`gh` effects occur (RS-003/codex testability rewrite)

### INV-010: Scoped, honest PR on success — diff verified, not self-reported
- **Type**: must · **Category**: security · **Risk**: high
- **Statement**: On a verified, regression-clean, CI-superset-clean fix, open exactly one PR for the selected issue; footer `Closes #{N}`. The PR scope is computed from `git diff {default}...HEAD` (the actual diff), NOT from `/cdebug`'s self-reported `files_changed[]` (TB-005 — verify, don't trust, RS-017). Before `gh pr create`, a **post-`/cdebug` diff allowlist check** aborts if the diff touches any SFG-protected path (catches an injection-driven mid-fix edit to `hooks/sensitive-file-guard.sh`/DEFAULTS that bypassed the pre-selection PRH-003 gate). Autonomous `/cdebug` Phase 5 (class-fix `antipatterns.md` write) is suppressed — class-fix assessment is deferred to human review and excluded from the chore diff (RS-017). PR/comment bodies are GENERATED from structured fields (INV-013), never a verbatim echo.
- **Violated when**: the PR bundles unrelated changes; scope is taken from `files_changed[]`; the diff touches an SFG-protected path; the PR claims to close issues it didn't fix.
- **Enforcement**: scoped staging allowlist (TB-004c pattern) + post-cdebug diff check + behavioral test · **Test**: integration

### INV-011: Fail-closed abort — persist first, durable marker, preserve evidence
- **Type**: must · **Category**: functional · **Risk**: critical
- **Statement**: On `/cdebug` `escalated`/`unfixable`/malformed-outcome, unsuitable issue, unverifiable fix, persistent regression, CI-superset failure, OR an SFG-protected target, `/cchores` ABORTS in this order: (1) write the full investigation to `.correctless/artifacts/chore-abort-{branch_slug}.md` (persist FIRST, AP-029; **gitignored, never pushed** — RS-018); (2) **record the abort in the local trusted re-selection store (INV-019) first** (so re-selection suppression does not depend on the public comment succeeding — RS-011/RS-018), THEN post a concise issue comment carrying a stable `<!-- cchores-abort -->` signature + the abort reason + the retained branch name (if any) + the resume steps (RS-021); the comment body passes INV-013 redaction. (3) branch cleanup — delete the chore branch ONLY if it has zero commits beyond base; if it has commits, RETAIN locally (do not push) and reference it in the comment AND surface it via `/cstatus` (INV-016, since a local-only branch is unreachable from GitHub — RS-021/codex); (4) record `status: aborted` + reason in the manifest. If any step fails mid-sequence, the run report records the partial-abort state and the local re-selection store still suppresses re-selection (the public comment is advisory, not the authority). No PR is opened.
- **Violated when**: an unverified fix is PR'd; an abort deletes a branch with useful commits; the artifact is not written before the comment; re-selection suppression depends solely on a public comment; a partial abort silently re-selects the same issue next run.
- **Enforcement**: ordered gate before PR; persist-before-present; local-store-before-comment; behavioral tests per trigger incl. branch-with-commits retention and comment-post-failure recovery · **Guards against**: AP-029; shipping unverified work; evidence loss; re-selection loops · **Test**: integration

### INV-013: Outbound redaction (coded, fail-closed) + caps — all egress sinks
- **Type**: must · **Category**: security · **Risk**: high
- **Statement**: All outbound text — PR **title** and body, issue comment, **commit message**, and **branch slug** (RS-007/codex: title/branch/commit are sinks, not just bodies) — is GENERATED from structured fields, then passed through a **NEW coded redactor `scripts/redact-secrets.sh`** (R3-2: `scripts/security-scan.sh` provides only PRH-001 category/keyword checks — `security_category_gate`/`security_keyword_scan` — NOT an outbound redaction API, so this is a new script, not a reuse). API contract: `redact-secrets.sh` reads text on stdin and writes redacted text on stdout, replacing each pattern match with `<REDACTED>`, exit 0 on success, non-zero (and empty stdout) if its pattern source is missing (fail-closed). **Pattern source (pinned, ordered)**: (1) the `/csetup` gitleaks config if present at `.correctless/config/gitleaks.toml`, ELSE (2) a bundled fallback `templates/secret-patterns.txt` (POSIX-ERE, one pattern per line, installed to `.correctless/config/secret-patterns.txt` by `setup`). It is the SOLE redaction entrypoint; `/cchores` pipes every outbound field through it (including the `/cdebug` structured outcome before any field is used) — NOT an LLM regex (RS-006). If the redactor is absent/non-executable or its pattern source is missing, `/cchores` **fails closed** (aborts — no posting); BND-002 preflight checks for it. PR body ≤ 8 KB, comment ≤ 4 KB, with overflow pointing to the (gitignored, local) artifact — and anything LINKED from a public comment is itself redacted (RS-018). The branch slug is additionally charset-constrained (INV-018).
- **Violated when**: a secret-shaped token or unbounded log reaches GitHub via any sink; redaction is LLM-performed; redaction fails open when the source is absent; any outbound field bypasses `redact-secrets.sh`.
- **Enforcement**: `scripts/redact-secrets.sh` (coded, fail-closed) invoked before any `gh`/`git push`/commit; `tests/test-redact-secrets.sh` enumerates EVERY pattern in the source and asserts `<REDACTED>` replacement, plus a behavioral test driving ≥3 canonical secret shapes through all four sinks and an absent-source fail-closed test · **Guards against**: secret egress to a public surface · **Test**: integration

### INV-015: Shared global working-tree lock (cchores ⇄ cauto)
- **Type**: must · **Category**: concurrency · **Risk**: high
- **Statement**: `/cchores` acquires a **shared global working-tree lock** at a FIXED, non-branch-scoped path `.correctless/artifacts/worktree.lock` before any selection or git operation, and releases it on every terminal path (success, no-op, abort, error). This lock is the mutual-exclusion point between any working-tree-mutating orchestrator. Because the existing `/cauto` lock is *branch-scoped* and cauto-specific (`scripts/cauto-lock.sh` `lock_acquire` is generic but `/cauto` uses a per-branch path with cauto-only messaging — R3-4), this feature ALSO updates `/cauto` to acquire the same global `worktree.lock` (in addition to its existing branch lock), and generalizes `cauto-lock.sh`'s error messaging to name the holding orchestrator. A second `/cchores` OR a concurrent `/cauto` that cannot acquire the global lock exits cleanly ("working tree is locked by {holder}"). The INV-004 final idempotency re-check runs under this lock (RS-010). Guards against two runs sharing the mutable working tree (AP-034).
- **Enforcement**: shared `worktree.lock` acquire/release via `scripts/cauto-lock.sh` `lock_acquire` (generic path arg) in BOTH `skills/cchores/SKILL.md` and `skills/cauto/SKILL.md`; stale-lock recovery test; a cross-skill mutual-exclusion test (cchores cannot acquire while a cauto-held global lock exists, and vice-versa) · **Test**: integration

### INV-016: Human-readable run report + /cstatus surfacing
- **Type**: must · **Category**: ux · **Risk**: medium
- **Statement**: On EVERY terminal state (success, no-op, abort), `/cchores` writes a human-readable run report `.correctless/artifacts/chore-report-{branch_slug}.md` (which issue + selection rationale, issues skipped + why, flake retries fired, outcome, PR/branch links) — the morning-after surface, analogous to `/cauto`'s Auto Run Report (RS-013). The BND-003 clean no-op also writes a manifest (`status: noop`) and a report so it is not a silent no-op. `/cstatus` is wired to read `chore-run-{branch_slug}.json` exactly as it reads `pipeline-manifest-*` (reporting `in_progress` truncation) and to surface retained abort branches (RS-013/RS-021).
- **Enforcement**: report write on every terminal path + `/cstatus` consumer wiring + test · **Test**: integration

### INV-017: Structural tool allowlist + runtime push-branch guard
- **Type**: must · **Category**: security · **Risk**: high
- **Statement**: `skills/cchores/SKILL.md` `allowed-tools` enumerates the FULL required command set (R3-5) and nothing broader than `Bash(*)`:
  - **gh** (subcommand-pinned, never `Bash(gh*)`): `Bash(gh issue list*)`, `Bash(gh issue view*)`, `Bash(gh issue comment*)`, `Bash(gh pr list*)`, `Bash(gh pr create*)`, `Bash(gh auth status*)`, `Bash(gh repo view*)` — so `gh pr merge`/`gh issue close`/relabel are structurally unreachable (RS-016/PRH-004; PAT-018).
  - **git**: `Bash(git status*)`, `Bash(git fetch*)`, `Bash(git switch*)`, `Bash(git reset*)`, `Bash(git restore*)`, `Bash(git rev-list*)`, `Bash(git ls-remote*)`, `Bash(git symbolic-ref*)`, `Bash(git diff*)`, `Bash(git add*)`, `Bash(git commit*)`, `Bash(git push*)`, `Bash(git branch*)`, `Bash(git remote*)`.
  - **tooling/scripts**: `Bash(jq*)`, `Bash(shellcheck*)`, `Bash(bash sync.sh*)`, `Bash(bash .correctless/scripts/redact-secrets.sh*)`, `Bash(bash .correctless/scripts/cauto-lock.sh*)`, `Bash(bash .correctless/scripts/autonomous-decision-writer.sh*)`, `Bash(bash .correctless/scripts/check-no-pending-sfg-lift.sh*)`, `Bash(timeout*)`, `Bash(gtimeout*)`, and the configured `commands.test` runner; `Task`; `Read`, `Grep`, `Glob`; `Write(.correctless/artifacts/*)`, `Write(.correctless/meta/cchores-attempted.json)`.
  - **Runtime push-branch guard**: `allowed-tools` glob `Bash(git push*)` **cannot** restrict push to the chore branch (R3-5 — a Bash pattern can't constrain the ref argument). So branch scoping is enforced at RUNTIME: before every `git push`, a coded guard asserts the target is `chore/issue-{N}-*` and HARD-fails on `main`/`master`/`develop`/`release/*` (the TB-004c protected-branch model, implemented as a check, not a glob).
- **Enforcement**: `allowed-tools` frontmatter (harness-enforced) for the command set + a runtime push-branch guard function + structural test asserting the pinned set AND a behavioral test asserting a push to a protected branch is refused · **Test**: structural + integration

### INV-018: Deterministic, charset-bounded branch slug
- **Type**: must · **Category**: security · **Risk**: high
- **Statement**: `{slug}` in `chore/issue-{N}-{slug}` is derived deterministically and constrained to `[a-z0-9-]` with a length cap (e.g. ≤ 40 chars), lowercased, collapsed dashes, no leading/trailing dash — NOT free-form LLM text from the issue title (RS-007). This prevents ref-injection / option-injection / namespace collisions when the slug flows into `git switch -c` and `--head`. The slug also passes INV-013 redaction.
- **Enforcement**: coded slug-derivation function + behavioral test feeding a hostile issue title · **Test**: integration

### INV-019: Cross-run re-selection store (ABS-044) — fully pinned
- **Type**: must · **Category**: data-integrity · **Risk**: high
- **Statement**: The authoritative loop-prevention store (R3-3 — promoted from OQ-006; it is load-bearing for INV-002/INV-011, not an open question) is a single file `.correctless/meta/cchores-attempted.json`:
  - **Path**: `.correctless/meta/cchores-attempted.json` (under `.correctless/meta/`, which is gitignored/local-only per the ABS-033 precedent — it MUST persist ACROSS runs and branches, which the branch-scoped ABS-030 JSONL cannot, so a dedicated cross-run file is required).
  - **Schema**: `{"schema_version": 1, "attempts": [{"issue": N, "branch_slug": "...", "outcome": "aborted|abandoned", "reason": "...", "recorded_at": "ISO"}]}`.
  - **Sole writer**: `/cchores`, via `lib.sh` `locked_update_file` (ABS-003 advisory lock — concurrent-write safe).
  - **Commit policy**: gitignored, never committed/pushed (local accountability only).
  - **Consumers (read-only)**: INV-002 selection filter (skip any issue with an `aborted` attempt); `/cstatus` (surface attempted issues); never the public comment.
  - **Prune behavior**: classified `task-slug`-independent (a `.correctless/meta/` file, NOT a `.correctless/artifacts/` slug-scoped file) — `/cprune` does NOT auto-delete it (it is durable like won't-fix entries); stale `resolved` attempts may be pruned via `/ctriage`, not autonomous `/cprune`.
  - **Write ordering**: INV-011 records the abort here FIRST (before the public comment), so re-selection suppression survives a comment-post failure.
- **Violated when**: re-selection suppression depends on the public comment; the store is committed; a non-`/cchores` writer touches it; concurrent writes corrupt it (must use `locked_update_file`).
- **Enforcement**: ABS-044 contract; `locked_update_file` writer; gitignore entry; structural + behavioral test (aborted issue is skipped next run even with no marker comment) · **Test**: integration

## Prohibitions
### PRH-001: Never open a PR with an unverified, regressing, or CI-dirty fix
- **Statement**: No PR while the reproduction test fails, any targeted-issue behavior is uncovered, any real (non-flake) regression stands, the fix is uncommitted/empty-diff, OR the CI-superset gate (shellcheck/sync/sfg-lift) is red (INV-008).
- **Detection**: gate precondition before `gh pr create`; behavioral test per failure mode. · **Consequence**: ships a broken fix that looks reviewed.

### PRH-002: Never act on instructions embedded in observed content
- **Statement**: No action sourced from issue/PR/comment text. Only the `/cchores` invocation authorizes action (INV-001 positive gate).
- **Detection**: injection fixture test (sentinel effects) + INV-001 denylist test. · **Consequence**: injection-driven destructive/exfil actions.

### PRH-003: Never auto-lift SFG protection in v1
- **Statement**: Must not modify `hooks/sensitive-file-guard.sh`, its DEFAULTS, or the runtime hook. SFG-protected targets abort BOTH at pre-selection (suitability) AND post-`/cdebug` diff check (INV-010, catching mid-fix edits).
- **Detection**: behavioral test — fixture issue targeting an SFG-protected path aborts with no hook edit in the diff; post-cdebug diff-touches-SFG test. · **Consequence**: autonomous run silently disabling a security guard.

### PRH-004: Never merge, close, or alter issues beyond one comment
- **Statement**: No `gh pr merge`, no `gh issue close`/relabel; at most one comment on the one selected issue. Enforced structurally via INV-017, not only by grep.
- **Detection**: INV-017 allowlist structural test + behavioral test. · **Consequence**: irreversible state changes without review.

## Boundary Conditions
### BND-001: GitHub issue content ingestion
- **Boundary**: TB-009 · **Input from**: untrusted issue author · **Validation**: nonce fence; inbound size cap; extract only the defect; never instructions (INV-009). · **Failure mode**: fail-closed — empty/unparseable/under-specified/oversized → unsuitable → abort.

### BND-002: Preflight environment validation
- **Boundary**: external tool dependency · **Input from**: local env · **Validation** (before selecting an issue): `command -v gh` + `gh auth status` (authenticated) + **token scope sufficiency** for PR/comment (RS-022); `git remote get-url origin` (GitHub remote exists); `command -v timeout`/`gtimeout` (EA-006); `test_fail_pattern` non-empty (INV-008); the coded redactor + secret-pattern set present (INV-013). · **Failure mode**: fail-closed — abort with a clear message naming the missing prerequisite; create no branch.

### BND-003: Empty candidate set · clean no-op (non-silent)
- **Boundary**: issue queue · **Validation**: if no suitable, non-in-progress, non-aborted issues exist (across full pagination, not just `--limit 100` — RS-028), write a manifest (`status: noop`) + a run report and exit cleanly. · **Failure mode**: clean, visible no-op.

## GitHub / Git Operations (exact)
- **Issue list**: `gh issue list --state open --limit 100 --json number,title,body,labels,createdAt` (paginate if 100 returned; do not assume completeness — RS-028).
- **In-progress check (exact ref, not substring)**: `gh pr list --state open --json number,headRefName,body` then match `.headRefName == "chore/issue-{N}-*"` OR `.body` containing `Closes #{N}`/`Fixes #{N}` — never a raw `{N}` search (RS-027). Plus `git ls-remote --heads origin "chore/issue-{N}-*"`.
- **Prior-abort marker**: the LOCAL re-selection store is authoritative (INV-011); `gh issue view {N} --json comments` grep of `<!-- cchores-abort -->` is a secondary signal only.
- **Remote/auth/scope**: `gh auth status`; `gh auth status` scope check; `git remote get-url origin`.
- **Default branch (cross-checked)**: `git symbolic-ref --quiet refs/remotes/origin/HEAD` AND `gh repo view --json defaultBranchRef --jq .defaultBranchRef.name`; disagreement/both-empty → abort (RS-020).
- **Worktree clean + ahead guard**: `git status --porcelain` empty; `git rev-list --count origin/{default}..{default}` == 0 before reset (RS-026).
- **Branch**: `git switch {default}` → `git fetch origin` → (ahead guard) → `git reset --hard origin/{default}` → `git switch -c chore/issue-{N}-{slug}` (slug per INV-018).
- **Scoped commit + push (R3-1/R4-1, cauto-style — INV-008 precondition, INV-010 scope)**: after `/cdebug` returns `fixed`, derive the stage set from the REAL working-tree state — `git status --porcelain` / `git diff --name-only` — NOT from `/cdebug`'s `files_changed[]` (which is used only as an advisory cross-check: if the real changed set diverges from `files_changed[]`, log the discrepancy and continue with the real set, TB-005 verify-don't-trust, R4-1). Every path in the real changed set must pass the INV-010 SFG/diff allowlist check (abort if any touches an SFG-protected path); `git add` exactly those paths (NEVER `git add -A`), then `git restore --staged .correctless/artifacts/ .correctless/meta/` as a belt-and-suspenders guard so manifest/abort/report/store artifacts never enter the commit (mirrors `/cauto` Step 8.2 / TB-004c). Then `git commit -m {redacted message}` (INV-013). The commit is the substrate INV-008 runs against (its `git diff {default}...HEAD` precondition). Push happens ONLY after INV-008 + the CI-superset gate pass: the runtime push-branch guard (INV-017) verifies the target is `chore/issue-{N}-*`, then `git push --set-upstream origin chore/issue-{N}-{slug}`. If `origin` has no upstream for the branch, `--set-upstream` creates it; a push failure aborts (INV-011) with the branch retained locally.
- **Re-verify open + PR**: re-check issue OPEN, then `gh pr create --base {default} --head chore/issue-{N}-{slug} --title {redacted} --body-file {path under .correctless/artifacts}` (body via file under the project dir, NOT `/tmp` — RS-022).
- **Comment**: `gh issue comment {N} --body-file {path under .correctless/artifacts}`.

## Design Decisions
- **DD-001**: Fully autonomous (no checkpoints) per user choice; blast radius bounded by fail-closed aborts, one-issue-per-run, run lock, no-merge, no-PR-on-failure, CI-superset gate, coded fail-closed redaction.
- **DD-002**: Compose, don't reinvent — Task-dispatch `/cdebug` (no `context: fork`; fresh context per step) BUT with INV-006's explicit per-path gating + agent migration + fail-closed parse-gate, because `/cauto` dispatches pipeline skills while `/cdebug` is the project's most interaction-dense skill — the `/cauto` precedent does NOT transfer for free (RS-002/codex). Reuse `autonomous-decision-writer.sh` (ABS-030 revised), `cauto-lock.sh` (INV-015), `build-caudit-prompt.sh` nonce fence (INV-009), the NEW `redact-secrets.sh` redactor (INV-013 — `security-scan.sh` is PRH-001 checks only, not a redactor), the `/cauto` PR shape, and the #161 table-anchor.
- **DD-003**: SFG-protected targets abort in v1, at BOTH pre-selection and post-`/cdebug` diff (INV-010); unblocked later by #176/#187.
- **DD-004**: Severity is calibrated LLM judgment (AP-028 triad), not label-trusted; the suitability gate + fail-closed aborts + explicit override + local re-selection store + audit log contain the risk. A deterministic label filter is intentionally NOT shipped (OQ-001 resolved: omit, not dead-key).
- **DD-005**: `/cchores` gets its OWN `chore-run-*` manifest (ABS-043), distinct from ABS-031. ABS-030 is REVISED (not just appended) to a script-level sole-writer with `cauto`+`cchores` invokers and an updated R-006d test (RS-004).
- **DD-006 (RS-009/RS-008)**: the regression oracle runs only against a COMMITTED, non-empty-diff chore branch, and the pre-PR gate is a CI superset — the "INV-008 green ⇒ PR ready" contract is only true if the gate the PR will face is a subset of the gate run before it.
- **DD-007 (R3-6, config migration — full surface)**: `patterns.test_file_marker` (default `""`) is added to BOTH `templates/workflow-config.json` (lite) and `templates/workflow-config-full.json`; `setup` migrates existing installed configs additively (adds the key with `""` if absent, never overwrites a set value — AP-004 idempotency); `/csetup` detection MAY populate it when a per-file test marker is detectable but defaults to empty otherwise; `docs/skills/cchores.md` documents it. The change is purely additive and namespaced under `patterns`, so `schema_version` is NOT bumped and existing configs without the key degrade per INV-008 (whole-suite blocking). No `chores` block ships in v1 (OQ-001), so there is no other new config surface.
