# Spec: AP-031 fixture divergence prevention

## Metadata
- **Task**: ap031-fixture-divergence-prevention
- **Recommended-intensity**: high
- **Intensity**: high
- **Intensity reason**: project floor (workflow.intensity = high)
- **Override**: none

## What

Structurally prevent AP-031 (test fixtures diverge from real producer output) by adding two-layer prompt-level directives to the spec and test writing phases. Layer 1: `/cspec` requires specs to pin the exact format being parsed when a feature consumes another Correctless tool's output, cross-referenced against the producer's SKILL.md or script. Layer 2: `/ctdd` RED agent requires at least one test fixture sourced from a real artifact in the repo rather than hand-written. A test audit check in `/ctdd` flags tests that parse skill/script output using only synthetic fixtures. A structural test verifies the directives exist. This addresses two back-to-back PMBs (PMB-010: sync script imported 0/25 findings; PMB-011: scanner produced 17 false positives) caused by the same class of fixture-format mismatch.

## Rules

- **R-001** [unit]: `skills/cspec/SKILL.md` contains a format-pinning directive integrated into Step 3 (Draft the Spec) stating: when a feature involves parsing output produced by another Correctless skill or script, the spec must (a) pin the exact format being parsed (heading regex, JSON schema, field names), and (b) cite the producer file path (SKILL.md template section or script path) as the authoritative format source. The directive must reference AP-031. The directive must include detection heuristics for the trigger condition: triggers when the feature reads, extracts from, or pattern-matches against files produced by another skill or script — including markdown heading parsing, JSON field access via jq, and regex matching against artifact content; does NOT trigger for file existence checks or path-only operations. The directive must include a concrete example: "Example: 'Heading format: `## Finding RS-{NNN}: {title}` per skills/creview-spec/SKILL.md Step 3.5 template.' Not: 'The script reads review findings.'" `Enforcement: prompt-level`
- **R-002** [unit]: `agents/ctdd-red.md` contains a real-fixture directive stating: when tests parse output from another Correctless tool (skill output artifacts, script JSON, meta files written by specific skills), at least one test fixture must be sourced from a real artifact in the repo. The preferred form is a verbatim excerpt included in the test file (or a tracked fixture file under `tests/fixtures/`) with a comment citing the source path — this form is hermetic and works in CI and fresh clones. Citation MUST use the prefix `# Source:` followed by the artifact path (e.g., `# Source: .correctless/artifacts/review-spec-findings-disallowed-tools.md`). The alternative form — reading the real artifact from its file path at test time — provides additional live coverage but must not be the sole form, since `.correctless/artifacts/` is gitignored and absent in CI or fresh clones. The directive must specify dormant behavior when no real artifact exists in the repo (new producer + consumer in same PR). `Enforcement: prompt-level`
- **R-003** [unit]: `skills/ctdd/SKILL.md` test audit section contains a fixture provenance check as check 11: for tests that parse output from another Correctless tool, flag as a BLOCKING finding any test suite that uses only inline heredoc or synthetic fixtures with no reference to a real artifact file. The check applies only to test files added or modified in the current feature branch; pre-existing tests are excluded. The `/ctdd` orchestrator computes the modified-test-file list via `git diff` against the base branch AND `git status --porcelain` for untracked files (RED phase creates new test files that are untracked, not just modified), and passes both lists to the test audit agent as input — the audit agent itself has read-only tools (Read, Grep, Glob) and cannot run git commands. The audit should also follow fixture file paths referenced by modified tests (e.g., `tests/fixtures/*.md`), not just examine the test files themselves. The check must distinguish between "no real artifact exists" (dormant — not a finding) and "real artifact exists but test doesn't use it" (finding). The directive must include a reference table of known producer-to-artifact patterns to support dormant detection (e.g., `/creview-spec` → `.correctless/artifacts/review-spec-findings-*.md`, `/caudit` → `.correctless/artifacts/findings/audit-*-round-*.json`). `Enforcement: prompt-level`
- **R-004** [unit]: A structural test file exists under `tests/` that verifies keywords appear within the correct section of each file (block-scoped, not file-wide — mitigates AP-003): (a) `skills/cspec/SKILL.md` Step 3 section contains: `"AP-031"` AND `"format"` AND `"producer"`, (b) `agents/ctdd-red.md` real-fixture directive section contains: `"real artifact"` AND `"# Source:"` AND `"dormant"`, (c) `skills/ctdd/SKILL.md` check 11 block contains: `"fixture provenance"` AND `"real artifact"` AND `"BLOCKING"` AND at least one concrete producer-to-artifact mapping (e.g., `"review-spec-findings-"`). Block-scoping: extract the relevant section (between heading markers or check number boundaries) before grepping, so keywords in unrelated sections don't satisfy the check. `Enforcement: structural (keyword-presence test)`
- **R-005** [unit]: Distribution copies in `correctless/skills/cspec/SKILL.md`, `correctless/skills/ctdd/SKILL.md`, and `correctless/agents/ctdd-red.md` match source copies after `sync.sh` runs. `Enforcement: structural (sync.sh + diff test)`
- **R-006** [unit]: `.correctless/AGENT_CONTEXT.md` test count is updated and the ctdd-red agent row or Design Patterns section references the AP-031 real-fixture requirement. `Enforcement: prompt-level`

## Won't Do

- Adding a script or hook to mechanically validate fixture provenance at test time — this is prompt-level enforcement, not tool-level
- Modifying existing test files to retroactively add real-artifact fixtures — this rule applies to new tests going forward
- Checksumming or head-diffing fixtures against source files — full checksumming is too brittle given artifact churn. The lighter variant (diff first N lines of real file vs fixture) is redundant when R-002 is followed: tests that read the real file at test time or include a verbatim excerpt with `# Source:` citation already exercise the real format directly. A head-diff alarm would fire on exactly the cases R-002 prevents. If the test reads the real file, format divergence is caught by the parser failing on mismatched content
- Promoting AP-031 to PAT-020 — the antipattern's own text says "if it recurs once more"; this spec is the prevention, not a third occurrence

## Risks

- **Prompt-level enforcement fades with context pressure**: The RED agent may ignore the real-fixture directive under high context load, especially on large specs with many rules. Mitigation: the test audit check (R-003) is a second-pass safety net that catches the omission. Accepted — two-layer prompt-level enforcement is proportionate to the risk (declarative skill output parsing, not security). PAT-018 deviation: all Layer 1/2 rules are prompt-level by design — structural enforcement would require runtime fixture validation, which is out of scope (Won't Do).
- **Both layers share a correlated failure mode**: Layer 1 (R-001, format-pinning in cspec) and Layer 2 (R-003, fixture provenance in test audit) both depend on recognizing that a feature parses another tool's output. If Layer 1 fails to trigger, Layer 2 is partially degraded because the spec contains no format citation for the test auditor to cross-reference. Mitigation: R-003's test audit fires independently of R-001 — the auditor examines test fixture content directly (presence of `# Source:` citations), not spec citations. The correlation is real but partial, not total. Accepted.
- **Real artifact may not represent current format**: Artifact files are generated once per workflow run and accumulate. The most recent artifact is the best proxy for current format, but older artifacts may reflect prior formats. Mitigation: R-001's format-pinning cross-references the producer's SKILL.md template, not the artifact — so the spec catches format changes even when the artifact is stale. Accepted.
- **Bootstrap deadlock for new producers**: A new feature that introduces both producer and consumer has no real artifact to reference. Mitigation: R-002 explicitly specifies dormant behavior for this case — the spec format pinning (Layer 1) is the sole guard, and the real-fixture requirement activates after the producer has run at least once. Accepted.
